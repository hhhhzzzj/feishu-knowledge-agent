# 企业知识整合与变更分发 Agent

基于飞书 OpenClaw、Lark CLI 与大模型能力构建的企业知识助手，面向企业办公场景提供两类核心能力：

- 多源企业知识整合与精准问答
- 知识变更感知与定向分发

## 项目背景

企业知识通常分散在飞书文档、Wiki、知识库等多个位置，员工在实际工作中常常面临两个问题：

- 想找知识时，检索成本高，定位慢
- 知识更新后，受影响的人无法被及时通知

本项目希望构建一个可理解、可沉淀、可主动服务的智能办公 Agent，在员工“提问时能答对”，在知识“变化时能主动推”。

## 核心能力

### 1. 企业知识整合

- 接入飞书文档、Wiki、知识库等多类数据源
- 对知识内容统一切片、索引、检索
- 支持跨文档问答与证据溯源

### 2. 精准知识问答

- 基于 Hybrid RAG 提升召回与回答准确率
- 支持返回来源文档、章节与原文片段
- 面向制度、流程、项目文档等场景进行问答

### 3. 主动变更分发

- 定时检测知识文档更新
- 自动提取关键变更点
- 基于部门、角色、标签识别受影响人群
- 向相关人员主动推送变更摘要与追问入口

## 典型场景

### 场景一：制度变更提醒

当《差旅报销制度》发生更新后，系统自动识别近期出差员工与相关部门成员，推送关键变化摘要，减少制度更新后的执行偏差。

### 场景二：日常知识问答

员工在飞书中直接向 Agent 提问，例如“差旅报销流程是什么？”，系统返回结构化答案及证据来源。

### 场景三：项目文档变更通知

当 API 文档或项目规范更新后，系统识别前端、测试、研发负责人等相关角色并定向通知。

## 技术方案

- 后端框架：FastAPI
- Agent 编排：LangGraph
- 检索方案：BM25 + 向量检索 + Reranker
- 向量模型：BGE-small-zh-v1.5
- 重排模型：BGE-reranker-base
- 向量存储：ChromaDB
- 状态存储：SQLite
- 飞书集成：OpenClaw / Lark CLI / Bot 通道
- 大模型：Doubao 2.0 Pro
